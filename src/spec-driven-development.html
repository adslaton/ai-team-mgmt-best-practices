The Perfect AI Specification Template

1. Context & Background
Provide the AI with essential context about the project, existing codebase, constraints, and requirements.
Context: Building a user authentication module for a React/Node.js SaaS application.
Existing: JWT-based auth system needs enhancement with OAuth2.
Constraints: Must maintain backward compatibility with existing API.
2. Clear Objectives
Define exactly what needs to be accomplished, with measurable success criteria.
Objective: Implement Google OAuth2 authentication
Success Criteria:
- Users can sign in with Google accounts
- Existing JWT tokens remain valid
- Response time under 200ms
3. Technical Requirements
Specify technical details, frameworks, patterns, and architectural decisions.
Tech Stack: Node.js 18+, Express 4.x, Passport.js
Architecture: RESTful API with middleware pattern
Database: PostgreSQL with existing user schema
Security: HTTPS only, CSRF protection required
4. Input/Output Specifications
Define exact inputs, outputs, data formats, and API contracts.
POST /auth/google
Input: { token: string } // Google ID token
Output: {
  success: boolean,
  user: { id, email, name },
  token: string // JWT
}
Error: { error: string, code: number }
5. Acceptance Criteria
Define specific conditions that must be met for the work to be considered complete.

‚úì Integration tests pass with real Google OAuth
‚úì Documentation includes setup instructions

Best Practices for AI Specifications

Clarity

Be Explicitly Clear

Remove all ambiguity from your specifications. What seems obvious to you may not be to the AI. Define technical terms, specify exact behaviors, and include examples wherever possible.
Scope

Define Boundaries

Clearly state what's in scope and what's out of scope. This prevents the AI from over-engineering or missing critical requirements. Include "NOT" requirements when necessary.
Examples

Provide Examples

Include concrete examples of inputs, outputs, and edge cases. Show the AI exactly what good looks like. Code snippets, API responses, and UI mockups are invaluable.
Constraints

State Constraints Early

Performance requirements, security constraints, compatibility needs, and coding standards should be stated upfront. Don't make the AI guess your non-functional requirements.
Testing

Include Test Scenarios

Specify test cases and expected behaviors. This helps the AI understand edge cases and ensures the implementation is robust. Include both happy path and error scenarios.
Iteration

Plan for Refinement

No spec is perfect on the first try. Build in checkpoints for review and refinement. Start with a minimal viable spec and enhance based on initial outputs.
